基于服务器解析并自动部署Hexo

发表于 2018-04-14 | 更新于 2018-04-24 | 分类于 技术 | 阅读次数：

基于GitHub双分支思路备份源码, 基于Git Hooks思路自动部署网页
客户端完成博客更新和配置修改, 服务器端解析并部署成静态网页
随时随地可以push更新到服务器, 服务器检测到更新后自动部署

准备工作

基于Github 双分支思路备份源码

  • 在Github Pages仓库中创建两个分支:master 和 hexo
  • 设置hexo为默认分支
  • 采用 Use SSH 方式将该仓库clone到本地
  • 将hexo文件夹中的文件复制到clone下来的git仓库中
  • 不要遗漏 .gitignore文件,并将 theme/next/目录下的.git/文件夹删掉
  • 将更新Push到git仓库

基于Git Hooks思路自动部署网页

  • 在云服务器中新建一个裸仓

  • 在裸仓的zingqi.git/hooks/文件夹目录下创建文件 post-receive,并写入

  • 给该文件添加可执行权限

  • 在hexo项目目录中,修改配置文件的deploy项目,使得每次deploy时将结果push到zingqi.git

本地一键更新

  • 服务器端编写脚本命令更新并部署Hexo

  • 服务器端利用Webhooks或者本地hook实现自动触发

    • 利用Github的Webhook功能(代码托管到Github)
      当有push请求的时候，触发Github的Webhook功能，到服务器端执行自动更新脚本
    • 利用本地hook功能呢(代码托管到私有服务器)
      当有更新的时候，触发本地git仓库的hook功能，执行自动更新脚本

  • 客户端可以是电脑客户端也可以是手机客户端
